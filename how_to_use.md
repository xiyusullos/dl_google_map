# 谷歌卫星瓦片下载器使用说明

## 概述

这是一个基于 Python 的 **谷歌卫星瓦片下载器**，具备以下核心特性：
- ⚡ 瞬时总数计算与实时 ETA 显示
- 🔄 边算边下并行模式，内存占用 O(1)
- 📊 流式分片入库，支持断点续传
- 🛡️ 超大范围防护（瓦片数量预警）
- 🚫 404 永久过滤机制

---

## 快速开始

### 1. 安装依赖

```bash
pip install aiohttp rich pyyaml
```

### 2. 运行程序

```bash
python main.py
```

---

## 配置说明

程序会自动读取 `config.yaml` 配置文件，若不存在则使用默认配置。

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `download.output_dir` | 瓦片输出目录 | `./google_tiles` |
| `download.concurrency` | 并发下载数 | `8` |
| `download.map_type` | 地图类型 (`s`=卫星) | `s` |
| `download.max_retries` | 最大重试次数 | `3` |
| `download.skip_existing` | 是否跳过已存在文件 | `True` |
| `download.chunk_size` | 数据库入库块大小 | `3000` |
| `download.fetch_batch_size` | 拉取批次大小 | `400` |
| `download.max_tiles_warning` | 瓦片预警阈值 | `10,000,000` |
| `network.proxy` | HTTP 代理（可选） | `None` |
| `network.timeout` | 请求超时时间（秒） | `15` |
| `db.path` | SQLite 数据库路径 | `./tiles.db` |

## 使用流程

### 主菜单

运行程序后显示主菜单：

```
🌍 谷歌卫星瓦片下载器 v3.4

[1] 🆕 新建下载任务
[2] 📋 管理/恢复历史任务
[3] 🚪 退出程序

请选择 [1-3]:
```

### 选项 1：新建下载任务

1. **输入 KML 文件路径**：程序会从 KML 文件中解析地理边界坐标
2. **输入下载级别**：用逗号分隔，如 `10,11,12`（范围 0-22）
3. **输入输出目录**：回车使用默认目录

#### KML 文件要求

KML 文件需包含坐标点，程序会自动提取 `<coordinates>` 标签中的经纬度信息。

**示例 KML 结构**：
```xml
<kml>
  <Document>
    <Placemark>
      <Polygon>
        <outerBoundaryIs>
          <LinearRing>
            <coordinates>
              116.0,39.0
              116.5,39.0
              116.5,39.5
              116.0,39.5
            </coordinates>
          </LinearRing>
        </outerBoundaryIs>
      </Polygon>
    </Placemark>
  </Document>
</kml>
```

### 选项 2：管理/恢复历史任务

查看所有历史任务列表，选择任务 ID 可恢复暂停的下载任务。

### 选项 3：退出程序

安全退出，当前任务会自动暂停。

## 操作说明

### 中断下载

按 `Ctrl+C` 可安全中断下载，任务会自动保存进度。

### 下载状态标识

| 标识 | 含义 |
|------|------|
| ✅ | 成功下载 |
| ❌ | 下载失败（会重试） |
| ⏭️ | 跳过（404 或已存在） |
| ⏳ | 待处理 |

## 技术细节

### 瓦片 URL 格式

```
https://mt0.google.com/vt/lyrs={map_type}&x={x}&y={y}&z={z}
```

### 地图类型参数 (`map_type`)

| 参数 | 地图类型 |
|------|----------|
| `s` | 卫星影像 |
| `m` | 标准地图 |
| `h` | 混合地图 |

## 注意事项

1. **合规使用**：仅限个人学习/科研使用，遵守 Google Maps 服务条款
2. **网络要求**：需要稳定的网络连接，建议使用代理访问谷歌服务
3. **存储空间**：高分辨率瓦片数量巨大，注意磁盘空间
4. **超大范围预警**：当预估瓦片数超过阈值（默认 1000 万）时会提示确认

---

## 数据库结构

程序使用 SQLite 数据库 `tiles.db` 存储任务和瓦片状态：

- **tasks 表**：存储下载任务信息
- **tiles 表**：存储每个瓦片的下载状态（1=待下载, 2=成功, 3=失败待重试, 4=永久跳过）
